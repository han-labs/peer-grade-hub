# Documentation

Use this directory for project documents that the team agrees to keep in version control.

- `api/`: API contracts, endpoint notes, and request/response examples.
- `class-diagram/`: UML class diagrams and mapping notes.
- `database/`: ERD, schema decisions, and migration notes.
- `meeting-notes/`: team meeting minutes and decisions.
- `srs/`: software requirements specification.
- `usecase/`: use case diagrams and use case descriptions.
